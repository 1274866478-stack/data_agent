## 使用指南

### 查看文档优先级
1. **新开发人员**: 先读`prd-v4.md`了解项目目标
2. **架构决策**: 参考`architecture-v4.md`和分片
3. **开发任务**: 查看`stories/`中的具体需求
4. **质量保证**: 参考`QA/`中的检查清单

- **变更记录**: 每个主要文档都有变更日志
